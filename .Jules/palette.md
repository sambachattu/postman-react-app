## 2026-06-25 - Add aria-labels to icon-only buttons
**Learning:** The project relies heavily on Lucide React icons within button elements. Without explicit `aria-label` attributes, these icon-only buttons are inaccessible to screen reader users, who will just hear "button". We should always ensure to include an `aria-label` for screen readers and a `title` for hover states on icon-only buttons. We also learned that updating `aria-label` can break existing test queries relying on an empty name attribute, so corresponding test updates are necessary.
**Action:** Consistently apply `aria-label` and `title` to all icon-only buttons moving forward. When modifying accessibility attributes on existing components, double-check test suites (e.g., `HeadersSection.test.jsx`) for tests relying on `getByRole('button', { name: '' })`.
